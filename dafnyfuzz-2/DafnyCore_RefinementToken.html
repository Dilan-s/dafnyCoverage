 67  /// The "RefinementTransformer" is responsible for transforming a refining module (that is,
 68  /// a module defined as "module Y refines X") according to the body of this module and
 69  /// the module used as a starting point for the refinement (here, "X"). In a nutshell,
 70  /// there are four kinds of transformations.
 71  ///
 72  ///   0. "Y" can fill in some definitions that "X" omitted. For example, if "X" defines
 73  ///      an abstract type "type T", then "Y" can define "T" to be a particular type, like
 74  ///      "type T = int". As another example, if "X" omits the body of a function, then
 75  ///      "Y" can give it a body.
 76  ///
 77  ///   1. "Y" can add definitions. For example, it can declare new types and it can add
 78  ///      members to existing types.

 82  ///      that in many other language situations, it's the original ("X") that says what
 83  ///      parts can be replaced. Here, it the refining module ("Y") that decides where to
 84  ///      "squeeze in" new statements. For example, if a method body in "X" is
 85  ///
